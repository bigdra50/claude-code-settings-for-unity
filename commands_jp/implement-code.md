$ARGUMENTS ファイルに記載された仕様を満たすコードを実装してください。

以下の手順に従ってください：

1. 仕様にテストケースが含まれていない場合は、このコマンドの実行を中止してください。
2. コンパイルできるプロダクトコードの型と公開メソッドシグネチャのみを作成してください。動作しなくても構いません。
3. ドキュメントのテストケースに基づいてテストコードを実装してください。
4. 追加したテストを実行して、失敗することを確認してください。
5. gitにコミットしてください。
6. プロダクトコードを実装してください。
7. `mcp__jetbrains__open_file_in_editor` と `get_current_file_errors` ツールを使用して、`error` の重大度レベルの診断を解決してください。
8. テストを実行して、すべてが合格することを確認してください。
9. gitにコミットしてください。
10. KISS と SOLID の原則を念頭に置いてリファクタリングを行い、テストが合格することを再確認してください。
11. `suggestion` 以上の重大度レベルの診断を解決し、テストが合格することを再確認してください。
12. `mcp__jetbrains__reformat_file` ツールを使用して、修正したファイルをリフォーマットしてください。
13. gitにコミットしてください。

注記：

- テスト実行についての情報は、@.claude/commands/run-tests.md ファイルを参照してください。

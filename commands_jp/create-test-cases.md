$ARGUMENTS ファイル内の仕様を検証するために必要なテストケースについて考え、ファイルに追加してください。

以下の手順に従ってください。

1. 仕様を読んで分析します。不明瞭な点がある場合は、このコマンドの実行を中止してください。
2. 統合度が最も低い順序で、テスト対象となるパブリッククラスとメソッドを選択します。
3. カバレッジに配慮したテストケースを導出するために使用するテスト技法（同値分割、境界値分析、状態遷移テストなど）を選択します。
4. テスト技法を使用して、自然言語でテストケースを作成します。以下の注意事項を守ってください。
    - 順序付きIDを決して作成しないでください。
    - 検証内容を説明します。検証できないテストケースは削除してください。
5. ステップ 2 から 4 を、テスト対象となるすべてのクラスがカバーされるまで繰り返します。
6. テストケースを $ARGUMENTS ファイルに追加します。
7. git にコミットします。

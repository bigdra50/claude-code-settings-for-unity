$ARGUMENTS ファイルの仕様に基づいて、Kent Beck の Canon TDD に従ってコードを実装します。

以下の手順に従ってください。

1. 仕様にテストケースが含まれていない場合は、このコマンドの実行を中止します。
2. コンパイル可能なプロダクトコードの型とパブリックメソッドシグネチャのみを作成します。動作していなくても構いません。
3. テストケースリストの最初から、「内部ステップ」に従って 1 つのテストケースのみを実装します。
4. ステップ 3 をすべてのテストケースが実装されるまで繰り返します。ループを抜けることはありません。

内部ステップ：

1. 単一のテストケースに基づいてテストコードを実装します。
2. 追加したテストを実行して、失敗することを確認します。
3. 追加したテストに合格するプロダクトコードを実装します。
4. `mcp__jetbrains__open_file_in_editor` と `get_current_file_errors` ツールを使用して、`error` 重大度レベルの診断を解決します。
5. テストを実行します。合格しない場合はコードを修正します。テストが 2 回連続で失敗した場合は、前のコードに戻りステップ 1 に戻ります。
6. KISS と SOLID 原則を念頭に置いてリファクタリングし、テストに合格するように再実行します。
7. `suggestion` 以上の重大度レベルの診断を解決し、テストが合格するように再実行します。
8. `mcp__jetbrains__reformat_file` ツールを使用して、変更したファイルをフォーマットします。
9. Git にコミットします。

注記：

- テスト実行に関する情報は @.claude/commands/run-tests.md ファイルを参照してください。
